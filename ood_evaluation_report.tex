\section{Scope}
This report summarizes out-of-domain (OOD) evaluation results from \texttt{final\_ood\_comparison.csv} and the generated confusion matrices.
All numeric values in this report are taken directly from the experiment output CSV; no values are manually edited.
\section{Split Construction and Leakage Control}
Splits were generated with \texttt{ood\_eval/build\_balanced\_ood\_splits.py}. The split policy used validation data for model-specific sources to avoid train-test leakage:
\begin{itemize}
\item \textbf{FMA real/fake}: sampled from Deezer validation split (\texttt{dataset\_medium\_split\_filtered.npy}, key \texttt{validation}).
\item \textbf{FMA fake encoder fairness}: fake samples were stratified across \texttt{encodec3, encodec6, encodec24, griffin256, griffin512, lac14, lac2, lac7, musika}.
\item \textbf{SONICS real/fake}: sampled from \texttt{SONICS/valid.csv}. Entries overlapping \texttt{SONICS/train.csv} are excluded.
\item \textbf{FakeMusicCaps fake}: sampled from the FakeMusicCaps pool.
\end{itemize}
\section{Evaluation Orchestration}
\begin{itemize}
\item \textbf{Deezer model}: evaluated with \texttt{deepfake-detector/scripts/eval\_manifest.py}, aligned to the original \texttt{eval.py} pipeline (configuration loading, preprocessing, architecture, weights).
\item \textbf{SONICS models}: evaluated with \texttt{sonics/benchmark\_pretrained.py} using the same CSV manifests.
\item \textbf{Metric consolidation}: confusion terms (TP, TN, FP, FN) and derived metrics were consolidated in \texttt{ood\_eval/final\_ood\_comparison.csv}.
\end{itemize}

\section{Interpretation Notes}
\begin{itemize}
\item On fake-only datasets, specificity (\texttt{spec\_real}) is undefined because there are no real samples.
\item On real-only datasets, fake recall is undefined because there are no fake samples.
\item Balanced accuracy is reported only when both class recalls are defined.
\end{itemize}
